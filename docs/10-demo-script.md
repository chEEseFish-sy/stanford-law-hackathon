# 演示脚本

## 目标听众

国内律所、法律支持团队、企业法务、创业公司财务或股权管理人员中的私密内测候选人。

## 开场说明

VeriCap 是一个 AI 辅助股权文件审计工作台。它帮助法律专业人员复核融资文件，抽取股权证据，重建 working cap table，并用来源证据提示不一致事项。它是复核辅助工具，不替代律师判断。

## 演示目标

展示产品如何从一组融资文件出发，更快形成带证据链接的复核结果，而不是只做泛化文档摘要。

## 演示流程

### 1. 从业务问题开始

说明：

- 融资文件里的 cap table 证据分散在多份文件中。
- 人工复核需要检查协议、决议、版本和表格。
- 错误可能影响尽调、估值、投资人信任和交割节奏。

### 2. 打开工作台

展示：

- 当前 case。
- 已上传文件列表。
- 文件处理状态。
- 当前 viewing version。

要传达的信息：

- 工作台围绕复核任务组织，而不是泛化聊天。

### 3. 查看文档

展示：

- 融资文件包。
- 文件类型识别。
- 来源高亮或证据片段。
- 如可用，展示前后版本对比。

要传达的信息：

- 系统首先把输出建立在文件证据上。

### 4. 展示抽取证据

展示：

- 投资人。
- 日期。
- 证券类型。
- 股份数量。
- 价格。
- 权利或审批事项。
- 来源引用和置信度。

要传达的信息：

- 关键字段都应可追溯。

### 5. 展示 Working Cap Table

展示：

- 草稿 cap table 行。
- 来源链接。
- 置信度或复核状态。

要传达的信息：

- 这是 working cap table，不是最终法律结论。

### 6. 展示冲突或复核队列

展示一个问题：

- 缺失证据。
- 投资人名称不一致。
- 日期不一致。
- 股份数量不一致。
- 版本冲突。

要传达的信息：

- 产品价值在于帮助用户找到真正需要复核的事项。

### 7. 请求 AI 解释

可以问：

- 为什么系统生成这行 cap table？
- 哪些证据支持这个冲突？
- 哪些内容需要律师复核？

要传达的信息：

- AI 解释必须引用证据并说明不确定性。

### 8. 结束时提出内测请求

询问测试者：

- 这是否能帮助你当前的复核工作？
- 哪个输出你信任或不信任？
- 真实使用前需要哪些数据安全前提？
- 是否愿意使用脱敏文件继续测试？

## 必须保留的免责声明

不要说 VeriCap 证明 cap table 在法律上正确。应说它生成一个带证据链接的 working view，供法律专业人员复核。

## 常见问题与回答

### 这是法律意见吗？

不是。它是给法律专业人员使用的复核辅助工具，最终仍需要律师判断。

### 可以上传真实客户文件吗？

首轮 Beta 建议使用样例或脱敏文件。只有在授权、数据处理、保留和删除规则明确后，才考虑真实文件。

### 这和普通聊天机器人有什么不同？

VeriCap 围绕证据抽取、事件重建、cap table 影响和冲突复核组织。聊天只是解释层。

### 这和 cap table 管理平台有什么不同？

cap table 管理平台通常假设录入数据已经正确。VeriCap 检查法律文件是否支持 cap table。
