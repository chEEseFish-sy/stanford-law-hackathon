# 用户画像

## 用户 1：创业公司融资律师

### 工作背景

负责 VC/PE 融资、公司设立、股权轮次、投资人权利、交割文件和尽调答复。经常在较短时间内复核多份交易文件。

### 工作目标

- 验证融资文件是否支持拟议 cap table。
- 快速发现数字、日期、主体、权利条款的不一致。
- 将结论追溯到具体条款和文件。
- 可以把第一轮整理工作交给 junior，但保留最终判断权。

### 痛点

- 证据分散在多份文件和多个版本中。
- 股权数字核对繁琐且容易出错。
- junior 的整理结果仍需要 senior 二次确认。
- 交易节奏快，难以做充分人工交叉核对。

### 对产品的期待

- 展示来源证据，而不是只给摘要。
- 优先提示高风险不一致事项。
- 区分事实、AI 推断和人工确认结论。
- 保留律师对最终判断的控制权。

## 用户 2：律师助理 / 法律支持人员

### 工作背景

准备文件包、追踪文件版本、整理交割材料，支持律师做尽调和交易复核。

### 工作目标

- 快速整理文件。
- 找出缺失或冲突文件。
- 为律师准备可复核的证据材料。
- 减少手工 Excel 对账。

### 痛点

- 文件命名和版本经常混乱。
- 关键证据埋在长文档里。
- 不容易判断哪些事项需要升级给律师。
- 手工抽取带来质量控制压力。

### 对产品的期待

- 每个文件有清晰处理状态。
- 有简单的复核队列。
- 证据片段能链接到具体文件。
- 能导出或分享复核笔记。

## 用户 3：企业法务 / 财务 / 股权管理人员

### 工作背景

维护公司股权记录，配合融资尽调，与外部律师、投资人和公司管理层协作。

### 工作目标

- 提前准备更干净的尽调材料。
- 对比内部 cap table 与法律文件。
- 在投资人或外部律师复核前发现缺口。
- 减少与外部律师之间的反复沟通。

### 痛点

- 历史股权记录可能与法律文件不一致。
- 老文件、审批文件或股权激励文件可能不完整。
- cap table 系统通常假设录入数据已经正确。
- 法律复核成本高、周期长。

### 对产品的期待

- 在正式尽调前做预检查。
- 输出有证据支持的不一致报告。
- 清楚区分系统建议与法律结论。
- 有明确的数据安全和保密说明。

## 用户 4：准备融资的创业者

### 工作背景

需要在融资前整理公司股权和历史文件。不是第一付费用户，但可能影响传播和试用。

### 工作目标

- 理解公司股权记录是否完整。
- 找到缺失审批或冲突文件。
- 给律师准备更好的材料。

### 对产品的期待

- 上传和问题摘要足够简单。
- 用通俗语言解释风险点。
- 明确提示仍需律师复核。
