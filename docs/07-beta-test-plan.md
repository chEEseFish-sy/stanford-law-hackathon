# Beta 内测计划

## 目标

验证 VeriCap 对国内法律专业人员复核融资文件和 cap table 证据是否有实际价值。

## 参与者画像

建议先邀请 3-5 位早期测试者：

- 融资、公司法或投融资交易律师。
- 熟悉交割文件整理的律师助理或法律支持人员。
- 可选：创业公司法务、财务或股权管理人员。

## 测试数据

建议按以下优先级使用数据：

1. 系统内置样例融资文件。
2. 团队自制的合成文件，并故意埋入不一致问题。
3. 参与者提供的脱敏文件。
4. 真实客户文件只能在明确授权并写清数据处理规则后使用。

## 测试形式

建议拆成两种测试形式：

- 自助上手测试：测试者尽量不经过讲解，直接进入产品并完成核心闭环。
- 引导演示测试：由主持人讲解业务背景和关键界面，再进入任务测试。

首轮建议优先验证自助上手测试，至少让部分测试者在几乎无提示情况下进入产品。

参考时长：

- 5 分钟：产品背景和免责声明。
- 10 分钟：自助上手观察或引导演示。
- 30-45 分钟：参与者任务测试。
- 15 分钟：访谈和反馈表。

## 必须说明的免责声明

VeriCap 是 audit copilot 和复核辅助工具，不提供最终法律意见，也不替代律师复核。所有输出都是带证据链接的草稿建议，需要人工验证。

## 测试任务

1. 打开产品。
2. 判断当前是否可操作。
3. 找到示例 case 或上传入口。
4. 打开 case 工作区。
5. 上传或查看一组融资文件。
6. 复核文件类型识别结果。
7. 查看抽取出的证据。
8. 查看 working cap table。
9. 将至少一个 cap table 字段追溯到来源证据。
10. 查看一个冲突、缺失或低置信度事项。
11. 判断该事项应确认、驳回还是待跟进。
12. 要求系统解释一个结果。
13. 对信任度、可用性和缺失工作流提出反馈。

## 指标

- 首次进入后理解第一步所需时间。
- 识别当前系统状态所需时间。
- 理解工作台所需时间。
- 定位某个 cap table 行来源证据所需时间。
- 发现的有用问题数量。
- 令人困惑或不可信的输出数量。
- 用户对证据链接有用性的评分。
- 用户对风险边界清晰度的评分。
- 失败后成功恢复比例。
- 是否需要主持人口头解释才能继续。
- 用户是否愿意继续使用真实或脱敏 matter 测试。

## 访谈问题

- 你们现在如何完成这类工作？
- 当前流程中哪一步最耗时？
- VeriCap 哪个输出最有用？
- 哪个输出你最不信任？
- 产品是否充分展示了不确定性？
- 你会在融资交易前还是交易中使用它？
- 哪些因素会阻碍你在真实客户项目中使用？
- 你的律所会提出哪些数据安全要求？
- 哪种导出或协作功能最有价值？
- 你更能接受按项目收费、团队订阅，还是尽调工具附加费？

## 成功标准

- 至少 3 位测试者完成核心流程。
- 至少 2 位测试者可在几乎无提示下完成进入、上传、查看证据、查看 cap table 的基本闭环。
- 测试者理解系统输出是复核辅助，而不是法律结论。
- 测试者能够将关键输出追溯到证据。
- 至少 2 位测试者能指出愿意继续测试的具体业务场景。
- 主要产品阻碍足够清楚，能指导下一轮开发。
