# VeriCap 产品简报

## 一句话定位

VeriCap 是面向融资律师和法律支持团队的 AI 辅助股权文件审计工作台。它从交易文件中重建 working cap table，将每个结果链接到来源证据，并提示需要律师复核的不一致事项。

## 产品类别

- 法律科技。
- 交易智能。
- 股权审计。
- 融资尽调工作流。
- 证据驱动的 cap table reconciliation。

## 目标用户

- 创业公司融资律师。
- 公司法、VC、PE、投融资法律团队。
- 律师助理、实习律师、paralegal 和法律支持团队。
- 高成长企业的法务、财务和股权管理人员。
- 准备融资尽调材料的创业者，作为次级用户。

## 核心问题

融资团队经常需要核对公司的 cap table 是否有法律文件支持。相关证据分散在增资协议、股东协议、投资人权利协议、董事会/股东会决议、股权激励文件、SAFE 或可转债文件以及多个历史版本中。人工复核耗时、容易遗漏，也难以在交易压力下形成可审计的证据链。

## 核心价值

- 降低融资文件整理和交叉核对时间。
- 将法律文件转化为股权相关证据和事件记录。
- 生成可追溯到来源文件的 working cap table。
- 提示冲突、缺失、低置信度字段和版本问题。
- 帮助律师优先复核风险点，而不是替代律师做最终判断。

## 差异化

VeriCap 不应被定位成通用法律聊天机器人，也不是泛化合同摘要工具。它的差异化在于从文档问答升级为证据驱动的交易智能：

- 从单文档摘要，升级为多文档股权事件重建。
- 从字段抽取，升级为时间线和 cap table 影响分析。
- 从 cap table 管理，升级为 cap table 可信度验证。
- 从黑箱 AI 回答，升级为来源证据可追溯的复核工作流。

## 当前原型资产

当前仓库已经包含：

- React/Vite 前端工作台。
- FastAPI 本地后端。
- DOCX 处理流水线。
- SQLite 工作台存储。
- 样例融资文件。
- 已处理的结构化 JSON 输出。
- 拓扑图、证据复核、事件时间线和 working cap table UI 模块。
- 产品、市场和技术设计资料。

## 产品边界

VeriCap 是 audit copilot，不提供最终法律意见，不判断交易是否合法有效，也不替代律师复核。

## 建议的首轮 Beta 切口

首轮 Beta 应聚焦创业公司融资文件包。最强初始场景是：上传一组融资交易文件，抽取股权证据，生成 working cap table，并通过来源证据复核冲突、缺失和低置信度事项。
