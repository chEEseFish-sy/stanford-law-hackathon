# Beta 内测范围

## 内测目标

验证 VeriCap 是否能帮助国内融资律师和法律支持团队更快复核创业公司融资文件，更清楚地追溯证据，并更早发现 cap table 不一致问题。

## 主要内测场景

法律专业人员上传一组融资交易文件。VeriCap 识别文件类型，抽取股权相关证据，重建 working cap table，识别冲突或缺失信息，并把所有关键结果连回来源证据，供律师复核。

## Beta v0.1 范围内

- 小范围私密内测。
- 按 case / matter 管理一组融资文件。
- 一键本地启动或受控演示环境入口。
- 首屏示例 case / 样例数据入口。
- DOCX 上传流程。
- 融资文件类型识别。
- 股权相关条款和事实抽取。
- 股权事件时间线。
- working cap table 快照。
- 关键字段来源证据展示。
- 冲突、缺失、低置信度提示。
- 人工复核状态。
- 清晰的系统状态与错误提示。
- 处理失败后的恢复路径。
- 基础操作日志。
- 使用样例文件或脱敏文件进行可控演示。

## 首轮支持的文件类型

- 股权购买协议 / 增资协议 / Stock Purchase Agreement。
- 投资人权利协议 / Investors' Rights Agreement。
- 表决权协议 / Voting Agreement。
- 优先购买权与共同出售协议 / ROFR and Co-Sale Agreement。
- 董事会决议 / Board Consent。
- 股东会决议 / Shareholder Consent。
- 股权激励计划修订文件。
- 期权授予或其他股权激励文件，如果存在清晰 DOCX 版本。

## Beta v0.1 范围外

- 最终法律意见。
- 自动判断交易是否可以交割。
- 完整企业级多租户权限系统。
- 复杂部署矩阵或开放式自助注册平台。
- 律所 DMS / iManage / NetDocuments 等系统集成。
- 扫描版 PDF OCR 全流程。
- 复杂跨境、多法域法律分析。
- 多轮融资完整历史重建。
- 覆盖所有合同类型。
- 未经律师复核自动生成正式法律 memo。
- 泛化协作平台或与核心复核无关的多页面产品扩张。
- 需要培训后才能使用的复杂工作流。
- 未经明确书面同意，将用户上传的法律文件用于模型训练。

## 部署假设

首轮私密内测优先采用本地、私有云或严格受控的演示环境。在权限、保留、删除、日志和模型服务商数据政策明确之前，不应开放公开上传真实敏感客户文件。

## 数据假设

默认使用样例文件、合成文件或脱敏文件。如果使用真实客户文件，内测参与者必须确认其有权提供文件，团队也必须明确数据保留和删除方式。

## 成功指标

- 新测试者在无人指导下能进入工作台。
- 新测试者能理解当前是否已连接后端、是否可上传、是否正在使用演示数据。
- 用户无需团队介入即可完成核心复核流程。
- 关键 cap table 字段能追溯到来源文件。
- 用户能理解哪些是 AI 生成、哪些需要人工复核。
- 用户至少能识别一个有用的证据、冲突或缺失问题。
- 处理失败能被看见和恢复，而不是静默失败。
- 用户在遇到失败时知道下一步该做什么。
- 内测用户愿意继续用真实或脱敏业务场景做下一轮测试。

## 明确非目标

第一轮 Beta 不是证明产品能覆盖所有法律 AI 场景，而是证明一个窄而高价值的工作流：基于证据链的融资文件 cap table 审计。
